## Coding Style

- React, Next.js, and TypeScript UI changes must follow `docs/engineering/coding-style.md`.
- Import ordering, formatting, nested ternary checks, block statement enforcement,
  and selected lint rules are owned by Biome.
- Apply the style guide to files you touch and directly surrounding code needed
  for coherence. Do not perform broad style-only rewrites in unrelated files.
- Prefer explicit project conventions when they conflict with the shared guide,
  and record intentional exceptions in the implementation summary or PR notes.
