## Storybook

- Follow `docs/engineering/storybook.md`.
- Add or update stories for reusable UI components and important changed states
  when Storybook is configured.
- Cover loading, empty, error, disabled, and meaningful interaction states when
  they are relevant to the change.
- Do not initialize Storybook, install Storybook dependencies, or write broad
  story coverage without explicit approval.
